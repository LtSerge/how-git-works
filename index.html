Values and Hashes

At its core, git is a map, which means its composed of keys and values
Values -> sequences of bytes, content of txt or binary file
you can give a value to git, and it will return a key
the key is the SHA1 hash
"Apple Pie" ---> 23991897e13e47ed0adb91a0082c31c82fe0cbe5 (unique, 1 hash for this string)
Sha1 hash are 20 bytes, example mac command to generate hash for applie pie:
echo "Apple Pie" | git hash-object --stdin
to make this command persistent, add a -w (write) flag:
the echo command to output content, and then pipe the result into hash object,
we also have to tell this object to get ints content from standard input
-every object in a git repo has a SHA1 hash, hash function takes a value, and returns a 20byte string key for that value
